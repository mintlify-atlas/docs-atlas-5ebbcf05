## Prerequisites

Before you begin developing for Regions Unexplored, ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Java Development Kit" icon="mug-hot">
    JDK 21 or higher is required for Minecraft 1.21.1 development
  </Card>
  <Card title="Gradle" icon="cube">
    Gradle is used for building the project (wrapper included)
  </Card>
  <Card title="Git" icon="code-branch">
    For version control and cloning the repository
  </Card>
  <Card title="IDE" icon="code">
    IntelliJ IDEA (recommended) or Eclipse with Gradle support
  </Card>
</CardGroup>

## Clone the repository

Start by cloning the Regions Unexplored repository:

```bash
git clone https://github.com/UHQ-GAMES-MODS/RegionsUnexplored.git
cd RegionsUnexplored
```

## Initial setup

<Steps>
  <Step title="Configure Gradle memory">
    The project is configured to use 4GB of RAM for Gradle. This is set in `gradle.properties`:
    
    ```properties
    org.gradle.jvmargs=-Xmx4G
    org.gradle.parallel=true
    ```
    
    <Tip>If you experience build issues, you may need to increase this value depending on your system.</Tip>
  </Step>

  <Step title="Generate IDE files">
    <Tabs>
      <Tab title="IntelliJ IDEA">
        Open the project directory in IntelliJ IDEA. The IDE will automatically detect the Gradle project and import it.
        
        Alternatively, use:
        ```bash
        ./gradlew idea
        ```
      </Tab>
      <Tab title="Eclipse">
        Generate Eclipse project files:
        ```bash
        ./gradlew eclipse
        ```
        
        Then import the project in Eclipse via **File → Import → Existing Projects into Workspace**.
      </Tab>
      <Tab title="Visual Studio Code">
        Install the **Extension Pack for Java** and **Gradle for Java** extensions, then open the project folder. VSCode will automatically detect the Gradle project.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Setup development workspace">
    Generate the Minecraft development environment:
    
    <CodeGroup>
      ```bash Fabric
      ./gradlew :fabric:21.1:runClient
      ```
      
      ```bash NeoForge
      ./gradlew :neoforge:21.1:runClient
      ```
    </CodeGroup>
    
    This command will:
    - Download Minecraft assets
    - Apply mappings and decompile Minecraft
    - Set up the mod development environment
    - Launch the Minecraft client with Regions Unexplored loaded
    
    <Note>The first run may take several minutes as it downloads and processes Minecraft.</Note>
  </Step>

  <Step title="Verify the setup">
    Once the Minecraft client launches:
    1. Check that "Regions Unexplored" appears in the mods list
    2. Create a new world and verify custom biomes generate
    3. Check the console for any errors or warnings
  </Step>
</Steps>

## Development workflow

### Running the game

<Tabs>
  <Tab title="Client">
    Launch the Minecraft client for testing:
    
    <CodeGroup>
      ```bash Fabric
      ./gradlew :fabric:21.1:runClient
      ```
      
      ```bash NeoForge
      ./gradlew :neoforge:21.1:runClient
      ```
    </CodeGroup>
  </Tab>
  
  <Tab title="Server">
    Launch a dedicated server for multiplayer testing:
    
    <CodeGroup>
      ```bash Fabric
      ./gradlew :fabric:21.1:runServer
      ```
      
      ```bash NeoForge
      ./gradlew :neoforge:21.1:runServer
      ```
    </CodeGroup>
  </Tab>
  
  <Tab title="Data Generation">
    Run data generators to create worldgen JSON files:
    
    <CodeGroup>
      ```bash Fabric
      ./gradlew :fabric:21.1:runData
      ```
      
      ```bash NeoForge
      ./gradlew :neoforge:21.1:runData
      ```
    </CodeGroup>
    
    Generated files are placed in `src/shared/21.1/main/generated/`
  </Tab>
</Tabs>

### Hot reloading

For faster development iteration:

1. Make code changes in your IDE
2. Use your IDE's "Build" or "Make" feature (Ctrl+F9 in IntelliJ)
3. The running game will hot-reload changed classes automatically

<Warning>Not all changes support hot-reload. Registry changes, mixins, and some initialization code require a full restart.</Warning>

## Project configuration

The mod uses the Cloche multi-loader build system. Key configuration files:

<AccordionGroup>
  <Accordion title="build.gradle.kts" icon="gear">
    Main build configuration defining:
    - Mod metadata (name, version, authors)
    - Dependencies (Biolith, Lithostitched)
    - Loader-specific configurations
    - Repository URLs
  </Accordion>
  
  <Accordion title="gradle.properties" icon="sliders">
    Build performance settings:
    - JVM memory allocation (4GB)
    - Parallel build execution
    - Kotlin code style
  </Accordion>
  
  <Accordion title="settings.gradle.kts" icon="screwdriver-wrench">
    Project name and plugin repository configuration for Cloche build system
  </Accordion>
</AccordionGroup>
